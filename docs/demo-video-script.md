# ClawTree 5–7 分钟演示视频脚本

参考：仓库根目录 [SPEECH.md](../SPEECH.md)。本脚本更适合正式录屏提交：每一段都包含画面动作、建议口播和对应评审打点。

建议录制时长：6 分钟左右。  
推荐入口：先运行 npm run demo，打开 http://127.0.0.1:3000。  
录制策略：默认使用离线 Demo，确保无数据库、无模型 Key、无邮箱、无钱包也能完整跑通；如果现场环境稳定，再补充展示 Django 或钱包真实能力。

## 0. 录制前检查

- 前端服务已启动：npm run demo。
- 浏览器准备 5 个标签页：
  - http://127.0.0.1:3000/
  - http://127.0.0.1:3000/demo
  - http://127.0.0.1:3000/user
  - http://127.0.0.1:3000/admin
  - http://127.0.0.1:3000/admin/proposals
- 如果网络不稳定，不依赖外部 API；$HTX 行情加载不出来也不影响主流程。
- 口播时强调：离线 Demo 不会真实发邮件，不会上真实链，不会泄露联系人。

## 1. 开场：问题与一句话定位（0:00–0:45）

画面动作：

打开首页 /，停留在 hero 区域，鼠标扫过 “AI × Web3 × MEDIA GROWTH” 和四步黄金路径。

口播：

大家好，我们是 ClawTree 团队。ClawTree 是为大树财经打造的 AI 媒体活动增长操作系统。

大树财经正在做全球高校行、AI/Web3 活动、黑客松和 Space，但今天媒体和高校合作有一个很真实的问题：信号很多，但行动很散。高校活动分布在官网、社团、公众号、活动平台；内容团队、运营团队和商务团队需要反复查证据、写方案、改邮件；赞助方最后看到的往往只是曝光数据，而不是可验证的执行链路。

所以我们做的不是一个活动黄页，也不是自动群发工具。ClawTree 要解决的是：如何把热点和高校信号，变成可信、可审核、可复盘的真实合作。

评审打点：

- 技术创新：把媒体增长拆成可编排、可验证的 Agent 工作流。
- 商业潜力：真实服务高校行、生态活动、赞助 campaign。
- 展示质量：第一句话明确产品定位。

## 2. 用户端：高校老师/学生看到什么（0:45–1:40）

画面动作：

切到 /user。展示首页三类入口：Signals、Events、Recaps。然后进入 /user/events，快速展示筛选、活动卡片、来源链接。再切 /user/recaps 展示活动回顾。

口播：

先看公开用户端。这一端面向高校老师、学生和活动参与者。用户不用自己翻 X 或到处找链接，就能看到大树财经整理后的校园信号、AI/Web3 活动和高校行回顾。

这里有一个重要边界：公开端只展示可公开的信息。活动来源、时间、主题可以展示，但联系人、内部评分、prompt、风险原文、邮件正文不会返回给用户。对财经、Web3 和高校合作来说，这个边界非常关键。

进入活动页，可以按 AI、Web3、AI+Web3 或活动类型筛选；进入回顾页，可以看到经审核后的内容摘要和来源。这不是单纯好看的前台，它是后面 campaign 增长闭环的公开承接页。

评审打点：

- 产品完成度：有真实用户端，而不是只有后台 Demo。
- UX 清晰：高校用户能发现活动、了解合作、查看回顾。
- 安全边界：公开 API 字段 allowlist。

## 3. 核心 Demo：从信号到 campaign（1:40–3:20）

画面动作：

切到 /demo。先展示 campaign 标题和信号列表。选择一个广州候选高校，例如暨南大学或港科大广州。展示匹配理由、公开证据链接、推荐合作形式。点击生成草稿。

口播：

现在进入核心 Demo：世界杯，或者更准确地说“全球足球赛事”，叠加广州高校行和 AI/Web3 主题。

ClawTree 会先把公开信号整理成 campaign：哪些内容是真实来源，哪些只是 AI 推断，哪些有合规风险。比如体育赛事热点很适合讲事件驱动市场、信息传播、概率校准和数据权益，但我们明确不做博彩、不预测比分、不提供投资建议、不承诺收益。

接下来系统匹配广州高校。这里每个目标都不是凭空生成的，它至少有公开证据：学校方向、学院能力、过往活动或相关研究。AI 给出推荐合作形式，比如公开课、AI Agent 挑战、数据权益圆桌或联合实验室。

现在点击生成外联草稿。可以看到它不是一封群发邮件，而是一校一稿：草稿里引用了 campaign 信号和学校证据，同时自动加入 guardrails：不博彩、不预测比分、不投资建议、不收益承诺、不结果保证，并且要求人工审批。

评审打点：

- AI 应用程度：AI 做分类、匹配、提案和草稿，不只是聊天框。
- 产品完成度：从 campaign 到目标高校到草稿有完整路径。
- 技术创新：来源约束、引用约束、合规约束同时进入 Agent 输出。

## 4. 人审外联与 Web3 proof（3:20–4:35）

画面动作：

在 /demo 点击人工批准/模拟发送，再点击生成 proof。停留在 proof payload、payloadHash、txHash、privacyFields 区域。如果页面展示 mock TRON proof，要明确指出是 mock。随后切到 /admin/outreach，展示审批台和 proof anchor 入口。

口播：

这里是 ClawTree 和普通自动化工具最大的区别：AI 不能直接发邮件，不能自动发布，不能自动上链。运营人员必须人工批准。

在 Demo 模式下，点击批准只会记录 simulated sent，不会真的发送邮件。然后我们生成一个 mock TRON proof。请注意链上或 mock proof 只包含公共摘要字段，比如 draftId、学校名称、活动标题、审批人、审批时间和审批状态。它不包含邮箱、联系人、邮件正文、回复原文，也不包含 prompt。

这个设计对应 Web3 的真实价值：不是为了“上链而上链”，而是为生态合作提供一个可验证的执行证明。赞助方、生态项目和主办方可以验证这个 campaign 确实经过审批、确实执行过，但看不到隐私数据。

评审打点：

- Web3 应用程度：链上 proof 解决可信执行和复盘问题。
- 技术难度：隐私 allowlist、确定性 hash、可扩展到 TRON Nile 合约。
- 商业潜力：赞助方 impact report、Grant 里程碑、生态合作凭证。

## 5. 运营后台：这不是单页 Demo（4:35–5:35）

画面动作：

切到 /admin。展示 Dashboard 指标，特别是 $HTX 卡片。如果行情加载成功，指出实时行情；如果没加载，说明这是 HTX 生态入口。再切 /admin/proposals 展示提案、引用证据和 Agent run；可快速切 /admin/ingestion 或 /admin/content 展示采集和审核状态。

口播：

再看管理端。ClawTree 不是一个单页演示，而是一个可运营的后台。

Dashboard 里有活动总量、AI/Web3 分类、待外联状态，也预留了 $HTX 行情入口，用来展示 HTX 生态集成。未来它可以扩展成 campaign 预算、赞助权益、生态积分或高校任务激励。

在提案页，我们可以看到三档合作建议：轻量媒体支持、中等内容共创、深度联合活动。每一档都要有证据引用和风险说明。对于技术评委，重点是 Agent 输出不是自由文本，而是有 schema、有 sourceIds、有 claim-level evidence、有 needsReview。

在采集和审核页，我们展示的是未来真实运营最需要的东西：来源、游标、失败、成本、审核状态和人审边界。只有这些都存在，AI Agent 才能进入真实组织流程，而不是停留在一次性 Demo。

评审打点：

- 产品完成度：公开端、管理端、Demo console 三端完整。
- 技术创新：Agent schema、引用覆盖、prompt injection 隔离。
- 生态契合：$HTX/HTX/B.AI 可以作为生态 campaign 工作台。

## 6. GENESIS / HTX 生态契合与商业化（5:35–6:25）

画面动作：

回到 README 或首页，也可以停留在 /admin Dashboard。展示项目名和核心口号。

口播：

最后讲一下为什么它适合 GENESIS 和 HTX 生态。

GENESIS 关注 $HTX 应用场景、B.AI 算力、AI Agent 金融、链上资产管理、DAO 工具和智能金融操作系统。ClawTree 的定位是生态增长与智能金融媒体操作系统。

对于 HTX、B.AI、开发者生态和 Grant 项目，ClawTree 可以把黑客松、校园活动、开发者招募和赞助 campaign 做成一个可追踪流程：AI 帮你发现机会、生成提案、辅助外联；人来审批关键动作；链上 proof 记录公共执行结果；最后输出 sponsor impact report。

商业化路径也很清晰：先服务大树财经广州高校行试点，再服务黑客松和生态项目的 campaign 工作台，进一步扩展为面向媒体、协议方、高校创新中心和赞助方的 SaaS。

评审打点：

- 商业潜力：从内部效率工具到 campaign SaaS。
- 生态契合：HTX / B.AI / TRON / Grant / DAO 协作都有落点。
- 投资叙事：把媒体活动从一次性传播变成可复用增长资产。

## 7. 收尾：一句话总结（6:25–6:45）

画面动作：

回到首页 hero，停在 “把热点和高校信号，变成真实合作”。

口播：

一句话总结：ClawTree 把热点和高校信号，变成真实合作。

它用 AI 解决活动增长的规模化问题，用人审解决品牌和合规风险，用 Web3 proof 解决生态合作里的可信执行问题。谢谢各位评委。

## 录制时最重要的 6 句话

1. ClawTree 不是活动黄页，也不是自动群发器，而是 AI 媒体活动增长操作系统。
2. 我们把“信号发现、证据核验、合作匹配、人审外联、链上凭证”做成完整闭环。
3. AI 只负责建议和草稿，发布、发送、上链都必须人审。
4. Web3 proof 只存公共摘要哈希，不存联系人、邮件正文和回复隐私。
5. 这对 HTX / B.AI / Grant / 黑客松生态的价值，是把活动增长变成可验证、可复盘、可规模化的 campaign。
6. 默认离线 Demo 可完整跑通，说明产品流程是稳定可演示的。

## 风险话术

如果评委问“有没有真实发邮件？”：

当前黑客松黄金路径默认使用 simulated sent，避免现场误发和外部副作用。真实 Django 原型已经保留审批发送边界，配置 SMTP 后才会真实发送。

如果评委问“有没有真实上链？”：

默认 Demo 展示 mock TRON proof 和隐私 allowlist；contracts/ 里有 EventRegistry、OutreachRecord、TrendOracle，可作为真实 TRON Nile 部署基础。我们更重视 proof 的数据边界：隐私不上链。

如果评委问“HTX 生态是不是只放了行情？”：

行情只是入口。真正的生态契合是把 HTX/B.AI/Grant/开发者活动做成 campaign OS：生态项目可用它发现高校合作、生成提案、跟踪外联、沉淀 impact report，并用 proof 记录公共执行结果。
