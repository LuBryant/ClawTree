# ClawTree 5–7 分钟 Demo 视频脚本

参考：仓库根目录 [README.md](../README.md)、[SPEECH.md](../SPEECH.md)、[PPT.md](../PPT.md)。  
这版脚本用于正式录屏提交：它不是功能清单，而是一条面向评委和投资人的产品叙事线。

一句话主线：

> ClawTree 把分散的热点、高校和 Web3 生态信号，变成可执行、可审批、可验证的真实合作。

推荐时长：6 分钟左右。  
推荐入口：运行 `npm run demo`，打开 `http://127.0.0.1:3000`。  
推荐策略：默认使用离线 Demo，保证无数据库、无模型 Key、无邮箱、无钱包也能完整跑通；如果现场环境稳定，再补充展示 Django 后端、SMTP 或真实钱包能力。

## 0. 录制前检查

录制前打开 6 个浏览器标签：

- `http://127.0.0.1:3000/`：首页，讲定位和黄金路径。
- `http://127.0.0.1:3000/demo`：核心端到端 Demo。
- `http://127.0.0.1:3000/user`：高校老师/学生视角。
- `http://127.0.0.1:3000/user/events`：公开活动列表。
- `http://127.0.0.1:3000/admin`：运营 Dashboard。
- `http://127.0.0.1:3000/admin/proposals` 或 `/admin/outreach`：证明后台不是摆设。

录制时注意：

- 保持浏览器缩放 90% 或 100%，让卡片、hash、guardrails 能完整入镜。
- 不要在视频里展示真实 `.env`、真实邮箱、真实 API Key 或真实钱包私钥。
- 如果 $HTX 行情没加载，不要停顿；说“行情入口已预留，核心流程不依赖外部 API”。
- 离线 Demo 不会真实发邮件、不会真实上链、不会触达真实联系人，这是安全设计，不是缺陷。

## 1. 开场：问题、市场和一句话定位（0:00–0:45）

网页上展示什么：

打开首页 `/`，停在 hero 区域。镜头扫过：

- “AI PARTNERSHIP INTELLIGENCE NETWORK”
- “Turn public signals into trusted partnerships”
- 三个 proof 指标：70%+、15 min、0 PII
- 首页四步黄金路径：发现可信信号 → 生成增长机会 → 人审外联 → 沉淀可验证结果

建议口播：

大家好，我们是 ClawTree 团队。

ClawTree 是面向媒体、高校、活动方和 Web3 生态的 AI 合作增长网络。今天我们用大树财经的高校行场景，展示一个非常现实的问题：全球事件、AI/Web3 热点、高校活动和赞助资源每天都在发生，但它们很难自动变成真正的合作。

传统做法是人工刷官网、翻 X、整理表格、写邮件、追回复。它慢、不可复用，也很难给赞助方证明“这次 campaign 到底做了什么”。而完全自动化又很危险：AI 不能直接代表品牌发邮件、发布内容或把隐私数据上链。

所以 ClawTree 的答案是一个半自动、可审计的增长操作系统：AI 负责发现、核验、匹配和草拟；人负责审批和承诺；Web3 proof 只记录公共摘要哈希，用来证明执行结果。

评审打点：

- 产品定位清晰：不是活动黄页，也不是群发器，而是 campaign growth OS。
- 技术判断成熟：AI 提效，人审控风险，Web3 做可信执行。
- 商业意义明确：服务媒体、高校、赞助方、Web3 生态项目的增长协作。

## 2. 用户端：高校老师和学生看到什么（0:45–1:35）

网页上展示什么：

切到 `/user`，展示公开入口的 Signals、Events、Recaps 三类卡片。  
再切到 `/user/events`，展示：

- 活动列表和 AI/Web3 分类标签。
- 搜索框、分类筛选、活动类型筛选。
- “联系信息仅管理端可见”或类似隐私边界提示。
- 活动详情/报名/来源链接。

如果时间够，切到 `/user/recaps`，展示活动回顾、来源、摘要、图片或展开全文。

建议口播：

先看公开用户端。这一端给高校老师、学生、开发者和活动参与者使用。用户不需要自己翻 X，也不需要到处找公众号和官网链接，就能看到大树财经整理后的校园信号、AI/Web3 活动和高校行回顾。

这里最重要的不是“页面好看”，而是边界清楚。公开端只展示可以公开的信息：活动标题、时间、地点、主题、来源和报名入口。联系人、内部评分、prompt、邮件正文和风险原文都不会返回给公开用户。

对财经、Web3 和高校合作来说，这个边界非常关键。ClawTree 不是把数据粗暴摊开，而是把可信信息变成一个可以承接流量、合作和报名的公开门户。

评审打点：

- 产品完成度：有真实用户端，不是只有后台或单页 Demo。
- 用户价值：学生发现机会，高校了解合作，媒体沉淀内容。
- 数据治理：公开 API 字段 allowlist，隐私和内部判断不外泄。

## 3. 核心 Demo：从热点信号到可执行 campaign（1:35–3:15）

网页上展示什么：

切到 `/demo`。按这条视觉顺序走：

1. 顶部标题：“全球足球赛事 × 广州高校行”。
2. Story rail：可信信号、主题设计、高校匹配、人工审批、影响凭证。
3. Funnel 指标：可信信号、增长机会、候选高校、正向回复。
4. Theme Package：公开课、Agent Challenge、Media Plan、Guardrails。
5. Signal Inbox：已核验信号、来源链接、发布时间、标签。
6. Opportunity Brief：AI 建议、目标、合作形式、引用数量。

建议口播：

现在进入核心 Demo。我们用“全球足球赛事 × 广州高校行 × AI/Web3”作为示例 campaign。

这个热点不是为了蹭流量。它可以被设计成一套真实的教育和传播产品：公开课讲事件驱动市场、信息传播和概率校准；Agent Challenge 让学生做数据分析和内容审核工具；媒体计划把一次活动拆成赛前预热、赛中讨论、赛后复盘、30 天回顾和 90 天总结。

注意这里的 guardrails：不博彩、不预测比分、不提供投资建议、不承诺收益、不保证结果。ClawTree 会把事实、AI 推断和合规边界分层展示。哪些是公开来源，哪些是 AI 建议，哪些必须人审，界面上都能看出来。

这就是我们认为 AI Agent 在真实组织里可落地的方式：不是让模型自由发挥，而是给它可验证来源、结构化任务和明确的安全边界。

评审打点：

- AI 应用程度：AI 做 campaign 设计、信号摘要、机会提炼，不只是聊天框。
- 媒体增长能力：一次热点被扩展成课程、挑战、传播和赞助资产。
- 合规意识：财经/Web3 场景主动规避博彩、投资建议和结果承诺。

## 4. 高校匹配与一校一稿：AI 负责建议，人负责决定（3:15–4:20）

网页上展示什么：

继续停在 `/demo` 的 Target Match 区域。展示：

- 5 所候选高校列表和 score。
- 点击一所高校，例如暨南大学、华南理工或港科大广州。
- 展示推荐共创形式、匹配理由、两条公开证据链接。
- 点击“生成外联草稿”。
- 展示草稿主题、正文、citationIds、guardrail checks。

建议口播：

接下来是高校匹配。系统不是随机生成学校名单，而是把 campaign 目标、大树财经能力库和高校公开证据放在一起计算。

比如选择一所广州高校，ClawTree 会显示推荐共创形式、匹配理由和公开证据链接。评委可以看到，每一个合作建议背后都有可追溯来源，而不是“AI 觉得合适”。

现在点击生成外联草稿。这里不是群发模板，而是一校一稿：草稿会引用 campaign 信号和学校证据，同时自动检查 guardrails。它会提醒我们不能写博彩、不能预测比分、不能做投资建议、不能承诺收益。

更关键的是，AI 到这里就停下了。它只能生成建议和草稿，不能代表团队真实发送。这一点对品牌、财经内容和高校合作都非常重要。

评审打点：

- 技术创新：sourceIds、claim-level evidence、guardrail checks 进入 Agent 输出。
- 产品实用性：从“找到目标”直接走到“可审草稿”。
- 品牌安全：AI 不越权，关键动作必须人审。

## 5. 人审外联与 Web3 proof：让执行结果可信，但不暴露隐私（4:20–5:15）

网页上展示什么：

在 `/demo` 的 Human Gate + Proof 区域：

1. 点击“人工批准并模拟发送”。
2. 展示 MOCK REPLY 和 suggested next step。
3. 点击“生成链上凭证（Mock Nile）”。
4. 停留在 proof 区域，让 payloadHash / network / privacy 提示完整入镜。

如果切后台，可打开 `/admin/outreach` 展示审批台、批准/驳回、钱包/合约入口；如果后端未连接，不强行演示，回到 `/demo` 即可。

建议口播：

这里是 ClawTree 和普通自动化工具最大的区别：所有真实副作用都要过人审。AI 不能自动发邮件，不能自动发布，不能自动上链。

在 Demo 模式下，点击批准只会记录 simulated sent，不会真的发送邮件。随后系统生成一个 mock TRON proof。请注意 proof 只包含公共摘要字段，比如 draftId、学校名称、campaign 标题、审批人、审批时间和审批状态。

它不包含联系人邮箱，不包含邮件正文，不包含回复原文，也不包含 prompt。链上记录的是公共摘要哈希，而不是隐私数据。

这就是 Web3 在这个项目里的现实意义：不是为了“上链而上链”，而是让赞助方、生态项目和主办方能够验证一个 campaign 确实经过审批、确实执行过、可以复盘，同时又看不到敏感信息。

评审打点：

- Web3 应用真实：proof 解决可信执行和 impact report，不是装饰。
- 隐私设计：链上只存公共摘要哈希，PII 和正文永不上链。
- 商业价值：可服务赞助验收、Grant 里程碑、生态协作记录。

## 6. 运营后台：证明这不是一次性 Demo（5:15–6:05）

网页上展示什么：

切到 `/admin`，展示：

- 活动总量、AI 活动、Web3 活动、待外联、$HTX 卡片。
- 快捷入口：活动浏览器、活动回顾、外联管道等。
- 系统状态区域。

再切到 `/admin/proposals`，展示：

- 三档合作提案：轻量合作、中度联动、深度合作。
- Required citations / Prohibited promises / masked email。
- Agent run evidence：input safe、injection quarantined、coverage、source IDs。

如时间足够，可快速切 `/admin/ingestion` 或 `/admin/content` 展示采集、审核、状态机。

建议口播：

再看运营后台。ClawTree 不是一个只能演一遍的单页 Demo，而是一个可运营的工作台。

Dashboard 展示活动总量、AI/Web3 分类、待外联状态，也预留了 $HTX 行情入口。今天它是生态入口，未来可以扩展成 campaign 预算、赞助权益、生态积分或高校任务激励。

在提案页，我们把合作分成轻量、中度、深度三档。每档都必须绑定证据引用和风险边界。这里还有 Agent run evidence：外部文本只进入不可信数据包，每个事实结论必须绑定允许的 source IDs；如果发现 prompt injection，会隔离，而不是让模型照着来源里的恶意指令执行。

这部分是给技术评委看的：我们没有把 Agent 当黑盒聊天机器人，而是把它放进一个有 schema、有引用、有状态、有审批、有测试的组织流程里。

评审打点：

- 产品完成度：公开端、Demo console、运营后台三端闭环。
- 工程深度：Agent schema、引用覆盖、prompt injection 隔离、审核状态。
- 可扩展性：可从大树财经工作区扩展到更多媒体、高校、生态项目。

## 视频里一定要说出的 8 句话

1. ClawTree 不是活动黄页，也不是自动群发器，而是 AI 合作增长操作系统。
2. 我们把“信号发现、证据核验、合作匹配、人审外联、链上凭证”做成完整闭环。
3. AI 只负责建议和草稿，发布、发送、上链都必须人审。
4. 每个合作建议都要有 sourceIds 和公开证据，缺证据就进入待确认。
5. 在财经和 Web3 场景里，我们明确禁止博彩、比分预测、投资建议、收益承诺和结果保证。
6. Web3 proof 只存公共摘要哈希，不存联系人、邮件正文、回复内容或 prompt。
7. 对 HTX / B.AI / Grant / 黑客松生态的价值，是把活动增长变成可验证、可复盘、可规模化的 campaign。
8. 默认离线 Demo 可完整跑通，说明产品流程稳定、可演示、无外部副作用。

## 风险问题话术

如果评委问“有没有真实发邮件？”：

当前黑客松黄金路径默认使用 simulated sent，避免现场误发和外部副作用。真实 Django 原型已经保留审批发送边界，配置 SMTP 后才会真实发送；没有人工审批不会发送。

如果评委问“有没有真实上链？”：

默认 Demo 展示 mock TRON proof 和隐私 allowlist。仓库 `contracts/` 里有 `EventRegistry`、`OutreachRecord`、`TrendOracle`，可作为真实 TRON Nile 部署基础。我们更重视 proof 的数据边界：隐私不上链，只锚定公共摘要哈希。

如果评委问“HTX 生态是不是只放了行情？”：

行情只是入口。真正的生态契合是把 HTX、B.AI、Grant、开发者活动和高校合作做成 campaign OS：生态项目可用它发现高校合作、生成提案、跟踪外联、沉淀 impact report，并用 proof 记录公共执行结果。

如果评委问“AI 会不会胡编学校或合作理由？”：

ClawTree 的设计是证据优先。每个事实结论都需要来源 ID 和引用覆盖；证据不足时降级为待确认，而不是直接进入外联。外部文本被当成不可信数据包，prompt injection 会被隔离。

如果评委问“为什么投资人应该关注？”：

因为这不是一个单点工具，而是一个可复制的生态增长基础设施。媒体、高校创新中心、协议方、交易平台、黑客松主办方和赞助商都有同一个痛点：如何把分散信号变成可信合作，并把结果证明给合作方看。
