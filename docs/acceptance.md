# ClawTree MVP 验收标准

本文件是交付检查表。每个 P0 功能必须能由命令、API 响应或清晰的人工步骤验证。

## A. Harness

- [ ] `npm run install` 在 Node.js 20.9+ 的干净环境成功。
- [ ] `npm run test` 通过数据、状态机和隐私字段测试。
- [ ] `npm run check` 通过 test、ESLint、TypeScript 和 production build。
- [ ] `npm run demo` 启动后 `/`、`/demo` 可访问。
- [ ] `npm run smoke` 自动走通 health → demo → draft → approve → anchor 并返回退出码 0。

## B. 产品主流程

- [ ] `/demo` 展示至少 4 条带原始 URL、发布时间、发布者和验证状态的信号。
- [ ] Demo 明确区分公开事实、AI 推断和 mock 数据。
- [ ] 至少 1 个 campaign 引用 3 条以上信号，并说明目标、受众和内容形式。
- [ ] 至少 3 个高校目标有 0–100 分数和两条可解释理由。
- [ ] `POST /api/outreach/draft` 返回 `subject/body/personalization/citationIds/guardrailChecks/status`。
- [ ] 草稿至少引用 2 条信号，guardrail checks 全部是布尔值。
- [ ] 未批准的草稿不得显示为 sent；批准后状态只能是 `simulated_sent`，并返回 `externalSideEffect:false`。
- [ ] 回复样例包含意图、置信度、摘要和下一步建议。
- [ ] 锚定结果包含 network、payloadHash、txHash 和明确的 mock 标识。

## C. 安全和可信度

- [ ] 所有 Demo 联系方式都显示 `MOCK`，不冒充真实高校联系方式。
- [ ] API 不执行真实邮件发送、公开发帖或链上写入。
- [ ] Proof payload allowlist 不含邮箱、联系人姓名、正文、回复原文或模型 prompt。
- [ ] 同一 campaign/draft 输入产生相同 payload hash。
- [ ] 页面不把 AI 生成内容描述为已确认合作。
- [ ] 金融内容包含“教育/活动策划，不构成投资建议”的边界。

## D. 体验和路演

- [ ] 评委从打开 `/demo` 到得到凭证不超过 3 分钟。
- [ ] 所有主要按钮有 loading、成功或错误反馈。
- [ ] 390px 和 1440px 视口无阻断性布局问题。
- [ ] 无网络、无模型 Key、无钱包、无数据库时黄金路径仍可运行。
- [ ] README 的命令、端口与实际实现一致。

## E. 可选增强（不阻塞 P0）

- [ ] Qwen/DeepSeek 适配器能在 10 秒内返回 schema 合法输出，失败自动降级 fixture。
- [ ] 至少一个真实只读信号连接器带缓存和来源追踪。
- [ ] 一次真实 TRON Nile 交易有浏览器链接，且 payload 通过隐私测试。
- [ ] 合约测试独立通过：`npm run install:contracts && npm run test:contracts`。

## F. TreeFinance Content Relay（候选，选中后才阻塞其工作包）

- [ ] 给定固定推文 fixture 连续运行两次，第二次新增数为 0，游标和运行记录正确。
- [ ] 每日任务能展示计划/实际时间、采集/新增/重复/失败数、耗时、模型成本和最后错误。
- [ ] 采集失败可重试并告警，不覆盖上一次成功游标。
- [ ] 10 条黄金样本的高校/AI/Web3/活动回顾分类达到团队约定阈值，并展示错误样本。
- [ ] tweet ID、URL/hash 和跨来源语义重复均有测试；被合并内容可追溯全部来源。
- [ ] AI 改写不覆盖原文；管理员可查看风险标签、建议版本、diff、模型版本和审核人。
- [ ] 未经编辑批准的内容不会出现在 `/user/recaps`。
- [ ] 公开回顾显示来源、原发布时间、本站更新时间和编辑状态；无授权媒体不复制展示。
- [ ] 内容处理是合规编辑而非规避平台规则，无法安全改写时允许 `rejected`。

## G. Campus Opportunity Radar（候选）

- [ ] 10 个黄金活动均保存来源、抓取时间、活动日期、报名状态和可信度。
- [ ] 过期、来源不明或日期冲突的活动默认不进入 `/user/events`。
- [ ] 联系邮箱必须有证据 URL、用途和最近核验时间；猜测邮箱样本被拒绝。
- [ ] `/user/events` 只展示活动与报名外链，不返回邮箱、电话或内部评分。
- [ ] `/admin/events` 能区分官方来源、活动平台和搜索发现，并进入人工核验队列。

## H. Collaboration Proposal 与外联（候选）

- [ ] “复旦机器人活动”样本输出契合点、三档合作包、待确认项、风险、置信度和事实引用。
- [ ] 每个契合点至少引用一条活动事实或大树能力库条目；无依据时明确为“待确认”。
- [ ] 选择 3 所学校生成 3 个独立 proposal 和 3 个独立消息对象，不产生拼接 BCC 正文。
- [ ] 未批准批次发送数始终为 0；批准后仍受每日上限、幂等键、退订和停止开关约束。
- [ ] 同一消息幂等重试不会重复发送；退订联系人永不进入待发队列。
- [ ] Demo 默认只模拟或创建草稿箱，不产生真实邮件副作用。

## I. `/user` 与 AI 客服（候选）

- [ ] `/user`、`/user/signals`、`/user/events`、`/user/recaps`、`/user/about`、`/user/cooperate` 的导航和空状态可演示。
- [ ] 高校老师无需访问 X 即可查看一条带来源的大树回顾、一个可报名活动并理解合作模式。
- [ ] 浏览器源码、网络请求和 production bundle 不含模型/邮箱/数据源 API Key。
- [ ] 客服回答显示依据或信息日期；未知问题转人工，不编造资源、嘉宾、奖金、曝光或投资承诺。
- [ ] 留资前明确用途并取得同意；用户端与管理端数据权限隔离。

## J. 世界杯 × 广州高校行主题包（候选）

- [ ] 主题能在 20 秒内解释为财经素养与 AI 数据实践，不被理解为博彩/荐股工具。
- [ ] 至少 3 所广州高校的匹配均有公开院系、实验室、课程或社团来源。
- [ ] AI 挑战评分包含引用覆盖、概率校准、反方证据和赛后复盘，不以猜中比分为目标。
- [ ] 世界杯名称、标识、图片和赛事数据均记录授权/使用规则；不明确时使用通用足球赛事表达与自有素材。
- [ ] 页面与提案不包含下注入口、币价预测、收益承诺或结果保证。
