大家好，我们是 ClawTree 团队。ClawTree 是面向媒体、高校、活动方和 Web3 生态的 AI 合作增长操作系统。

今天我展示的不是一个静态网站，而是一条 Live Workflow —— Django 后端、AI Agents 自动采集、AI 智能外联、人工审批、链上存证，贯穿一条完整路径。

我们要解决的问题很现实：高校活动、Web3 热点、赛事话题、开发者活动和赞助资源每天都在发生，但它们分散在官网、X、公众号、活动平台和社群。人工刷信息、整理表格、逐项写邮件、追踪回复，效率很低，也很难向赞助方证明 Campaign 真实执行过。

ClawTree 的内核就是这条链路：发现 → 聚合 → 智能外联 → 审批 → 链上凭证。AI 负责规模化，人审负责品牌和合规，Web3 Proof 负责可验证执行。

先看运营后台，这里的活动统计、AI 采集和待处理链数量来自 Django API，不是写死在页面里的数字。Django 后端负责承载真实运营数据：高校活动、活动回顾、采集运行、外联草稿、审批状态和 Proof 记录。前端只是操作台，真正的业务状态在后端。

这里是 ClawTree 的第一个内核能力：AI Agent 自动采集与聚合。传统团队需要人工到高校官网、社群平台、活动平台和 X 上找线索。ClawTree 把这个过程拆成可监控的流水线：先采集高校 AI/Web3 活动，再抓取和筛选大致相关内容，然后进入邮件草稿、审核和审批流程。这不是简单爬虫，真正重要的是运营可靠性——每一步都有日志、定时运行状态、失败记录和人工介入。对这些组织来说，AI Agent 不能只会跑一次，它必须能够启动、停止、审计、复盘。

在采集聚合之后，公开前端承接的是安全、可公开的信息。高校老师、学生和开发者可以看到活动标题、时间、地点、主题、来源和报名入口。但公开端不会返回联系人、内部评分、Prompt、邮件正文、原始原文和外联状态。搜索框也接入了 AI 自然语言理解能力，用日常描述就能智能检索出匹配的活动。X Space 的语音节目也接入了 AI 语音转文字总结，可以直接查看详细概览，省去人工逐场收听的时间。ClawTree 不是把数据粗暴摊开，而是把同一份活动数据按角色分层：公开端服务发现，管理端服务合作，Proof of Work 服务可信复盘。

现在展示第二个内核能力：AI 智能外联。活动浏览器里，运营团队可以看到后端采集的高校活动和公开联系信息，点击 AI 生成邮件。Django 后端会读取活动信息、工作区配置和活动的内容库，生成一封合作邀请草稿。

这里有两个关键点：第一，它不是群发模板，而是一校一稿——AI 会根据活动主题、学校和合作场景，生成个性化内容。第二，它不会直接发送，生成后的邮件只进入外联审批队列。"批量"也只代表批量生成独立草稿，不代表一键群发，不代表自动触发。

这就是 ClawTree 的产品哲学：AI 负责把增长动作规模化，人负责最后的品牌承诺。

这里是 ClawTree 的安全闸门：外联审批台。AI 生成的每一封邮件，都必须逐校逐封人工审核。运营人员可以修改正文、批准或驳回。没有人工审批，系统不会发送。今天本地默认使用 Django Console Email Backend，所以点击批准不会误发给真实高校，而是打印在后端终端——这证明真实 Django 发送路径已经接好，同时避免现场产生不可控的副作用。在生产环境里，这里可以接 SMTP 或第三方 Email Provider，但仍然必须保留逐封审核、人工审批。

最后是链上存证，也是 ClawTree 和普通 CRM、自动化工具最大的区别。当外联经过人工审批后，系统可以生成链上凭证。这个 Proof 的价值不是为了 Web3 而 Web3，而是给 Sponsor、Grant 生态项目和主办方一个可验证的执行记录：这次 Campaign 什么时候生成、什么时候审批、对应哪个活动、状态是什么。但隐私不上链——联系人、邮箱、邮件正文、回复原文和 Prompt 都不会进入链上 Payload，链上只锚定公共摘要和哈希。

刚才我们展示的是 Django 每个环节——采集、聚合、外联、审批、Proof 都接到了真实后端。

现在用 Demo 把我们最关心的一条端到端路径串起来：从全球热点和高校信号出发，设计成 AI Web3 校园 Campaign，匹配目标高校，生成一校一稿，经过人工批准，最后生成 Privacy Safe Proof。

为什么这个项目有投资价值？黑客松、协议方和高校创新中心都需要做同一件事：找到开发者、学生、社群和媒体合作伙伴，把活动从一次性传播变成可复用的增长资产。ClawTree 可以成为这类生态增长的操作系统：AI Agent 帮你发现和聚合机会，AI 智能外联帮你优化触达，人审确保品牌和合规，链上 Proof 让 Sponsor 和生态方验证结果。

最后总结：ClawTree 把分散的高校、媒体和 Web3 的生态信号链成真实合作。我们用 AI Agent 自动采集解决发现问题，用 AI 智能外联解决规模化问题，用人工审批解决品牌和合规风险，用链上存证解决可信执行和复盘问题。
