## 第 2 页 · 故事起点

**标题**：世界杯来了，高校能做什么？

**要点**：
- 世界杯不只是比赛——信息传播、概率校准、AI 决策、数据权益、体育消费、内容出海
- 树爪智动"全球高校行"要抓住这个全球事件，联动广州高校
- 但一个活动从策划到执行到后续传播，运营成本极高
- ClawTree 把一次活动扩展为五波内容：赛前预判 → 赛中直播 → 赛后复盘 → 30 天回顾 → 90 天总结

**配图**：世界杯 + 广州高校地图

## 第 3 页 · 痛点与方案

**标题**：规模化运营需要 AI Agent 引擎

**要点**：
- 高校活动信息分散在官网/公众号/社团，人工收集效率低
- 外联逐校发邮件耗时耗力，难以覆盖全国
- AI 不能群发、不能自动发布——需要来源可追溯、AI 可解释、人工可审批
- ClawTree 方案：发现 → 聚合 → 智能外联 → 人审 → 链上凭证

**配图**：闭环流程图（5 步）

## 第 4 页 · 高校资源孤岛 — 树爪智动是连通器

**标题**：有好活动、好项目，但别的学校不知道

**要点**：
- 各高校 AI 实验室、黑客松、讲座、竞赛资源分散，校际信息极度不对称
- 清华的夏令营、浙大的 AI 创意赛、北大的黑客松——同一个生态却互不相通
- 老师和学生缺少了解"别校在做什么"的窗口，合作机会被物理距离和信息壁垒切断
- ClawTree 把全国高校活动聚合到同一平台，树爪智动成为高校间的"活动路由器"
- 跨校学生社团联动、嘉宾资源共享、AI 项目协作——都从"看见"开始

**配图**：多校 logo 从分散到连通的示意图

## 第 5 页 · 六大 Agent

**标题**：六大 AI Agent 分工协作

**要点**：
1. Content Relay Agent — 每日增量采集树爪智动公开内容，分类去重合规编辑
2. Campus Scout Agent — 发现高校 AI/Web3 活动，校验日期、来源、联系方式
3. Collaboration Match Agent — 活动事实 × 大树能力库，生成三档合作提案
4. Outreach Copilot — 逐校生成独立草稿，批量只代表批量生成和审批
5. Reply Triage Agent — 分类回复意图，提取待办建议
6. TreeFinance Service Agent — AI 客服，审核知识库回答，不擅自承诺资源

**配图**：Agent 协作示意图

## 第 6 页 · 用户端

**标题**：高校老师/学生看到什么

- 公开端不展示联系邮箱和电话
- AI 客服了解合作方式和资源边界，高意向转人工

## 第 7 页 · 运营端 — 活动浏览器 + 外联审批

**标题**：运营团队的指挥中心

**要点**：
- 活动浏览器：OpenClaw 自动采集，真实联系方式，AI 生成合作邮件
- 外联审批台：逐校逐封人审，批准后 SMTP 真实发送
- 链上凭证：TronLink → OutreachRecord 合约 → TRON Nile 上链
- 链上只存 ID、高校名、邮件哈希——不含联系人和正文

## 第 8 页 · 自动化流水线

**标题**：4 步自动化，定时·上限·启停

**要点**：
- 08:00 活动采集：OpenClaw → JSON → 自动入库
