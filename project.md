# ClawTree（树爪智动）项目简述

> 本文件是对外快速介绍；详细范围、现状和待决策项以 [docs/prd.md](docs/prd.md) 为准。

## 一句话简介

ClawTree 是基于 AI Agent 的 Web3+AI 高校生态协作引擎：把大树财经与高校散落的公开内容和活动信号，转成可审核的活动回顾、校园机会、合作提案与人审外联，让老师和学生更容易发现活动，也让大树财经更高效地连接高校。

## 为什么做

大树财经正在推动全球高校行、AI/Web3 活动、黑客松和嘉宾分享，但信息与行动仍被割裂：

- 大树的高校内容散落在 X、公众号等渠道，国内高校老师和学生查找、访问和筛选成本高。
- 高校 AI/Web3 活动分散在校官网、学院、创新中心、社团和活动平台，人工检索容易遗漏。
- 即使找到活动和公开合作邮箱，运营仍要逐校研究契合点、写提案、改邮件和跟进回复。
- 高校方往往不了解大树能提供的媒体、嘉宾、Space、黑客松与生态资源，合作需要多轮重复沟通。
- 全自动群发和自动发布会损害品牌；真正需要的是来源可追溯、AI 可解释、人工可审批的半自动流程。

## 产品双端

### 用户端 `/user`（规划中）

面向高校老师、学生和活动参与者：

- 浏览经编辑审核的高校/AI/Web3 热点和“校园信号”。
- 查看大树财经过往高校行、活动照片/摘要和演讲回顾，无需自行访问 X。
- 发现各高校近期活动，查看时间、来源和报名入口。
- 通过 AI 客服了解大树财经、合作方式和资源边界，高意向问题转人工。

### 管理端 `/admin`（已有部分原型）

面向大树财经运营、编辑和生态团队：

- 查看每日采集任务、来源、失败和增量游标。
- 审核 X/公众号内容的主题分类、跨源去重、合规风险和客观改写建议。
- 检索高校活动与带证据的公开合作邮箱。
- 比较高校活动与大树能力，生成有引用的三档合作提案。
- 为每所学校生成独立邮件草稿，批量进入人工审批后再限速发送。
- 解析回复意图，形成合作漏斗和可验证影响报告。

## 核心 Agent

1. **Content Relay Agent**：每天增量获取 TreeFinance 公开内容，分类、去重、合规编辑并进入人审发布队列。
2. **Campus Scout Agent**：发现近期高校 AI/Web3 活动，校验日期、来源、报名状态和公开联系方式。
3. **Collaboration Match Agent**：把活动事实与大树已审核能力库对齐，输出契合点、缺失信息、风险和三档提案。
4. **Outreach Copilot**：生成逐校草稿；批量只代表批量生成和审批，不代表无人值守群发。
5. **Reply Triage Agent**：分类回复、提取待办并建议人工下一步。
6. **TreeFinance Service Agent**：以审核知识库回答高校问题，不知道时转人工，不擅自承诺资源。

## 世界杯 × 广州高校行

首个有传播力的主题包不做比分预测或博彩，而把世界杯视为“全球事件驱动市场实验室”：围绕信息传播、概率校准、AI 决策、数据权益、体育消费和内容出海，组织广州高校公开课、AI Agent 赛前叙事/赛后复盘挑战，并将一场活动扩展为赛前、现场、赛后、30 天和 90 天五次内容传播。

## 当前代码基线

- 已有 Next.js 离线 Demo/harness、Dashboard 与 mock 黄金路径。
- 已有 Django 高校活动采集、TreeFinance X 推文采集、DeepSeek 筛选/摘要/风险处理、80% 语义去重及管理 API。
- 已有 `/admin`、活动浏览、活动回顾和 AI 客服原型。
- 尚未实现独立 `/user`、每日推文调度、公众号合规适配、结构化合作提案和可靠邮件审批发送。

本轮只完成产品与工程前期设计，不实现上述新增功能。所有候选功能都已拆入 [docs/tasks.md](docs/tasks.md)，只有通过决策门后才进入开发。

## 安全与可信边界

- AI 不自动公开发布，也不跳过人工审批发送邮件。
- “去敏”指风险识别、客观改写建议和编辑审核，不用于绕过规则或改变事实；原文和 diff 对管理员可见。
- 只采集带来源的公开机构/活动联系方式，不猜测个人邮箱；支持限速、幂等、退订和抑制。
- 公开用户 API 不返回联系邮箱、内部评分、未发布内容或原始风险文本。
- 模型和邮箱密钥只能保存在服务端；链上只存公共摘要哈希，不存联系人或邮件正文。
- 世界杯内容不提供博彩、投资建议、收益承诺或结果保证。

## 技术栈

- 前端与离线 Demo：Next.js 16、React 19、TypeScript。
- 真实原型后端：Django、Django REST Framework。
- 模型：Qwen 百炼与 DeepSeek，通过服务端 OpenAI-compatible gateway 调用。
- 可选 Web3：Solidity/Hardhat，链上只做隐私安全的公共摘要凭证。
- 数据源：X API/合规服务、高校官网、授权公众号输入；所有来源保留 URL、时间与内容哈希。
